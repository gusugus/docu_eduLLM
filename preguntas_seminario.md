# Banco de Preguntas Completo – Ingeniería de Sistemas

## 📘 Programación Orientada a Objetos (POO) – Preguntas 1 a 30

### Pregunta 1
**¿Qué es la Programación Orientada a Objetos (POO)?**  
✅ Respuesta: Paradigma basado en objetos que contienen atributos y métodos.  
📌 Explicación: La POO organiza el software mediante clases y objetos para facilitar reutilización y mantenimiento.

### Pregunta 2
**¿Qué es una clase?**  
✅ Respuesta: Una plantilla para crear objetos.  
📌 Explicación: Define atributos y métodos que tendrán los objetos creados a partir de ella.

### Pregunta 3
**¿Qué es un objeto?**  
✅ Respuesta: Una instancia de una clase.  
📌 Explicación: Es un elemento real creado a partir de una clase.

### Pregunta 4
**¿Qué es encapsulamiento?**  
✅ Respuesta: Ocultar datos internos y controlar el acceso.  
📌 Explicación: Protege los atributos usando modificadores como `private`.

### Pregunta 5
**¿Qué es herencia?**  
✅ Respuesta: Permitir que una clase herede atributos y métodos de otra.  
📌 Explicación: Facilita reutilización de código.

### Pregunta 6
**¿Qué es polimorfismo?**  
✅ Respuesta: Permitir diferentes comportamientos para un mismo método.  
📌 Explicación: El método cambia según el objeto que lo implemente.

### Pregunta 7
**¿Qué es abstracción?**  
✅ Respuesta: Mostrar solo lo esencial ocultando detalles internos.  
📌 Explicación: Reduce complejidad y simplifica el uso de objetos.

### Pregunta 8
**¿Qué hace el modificador `private`?**  
✅ Respuesta: Permite acceso solo dentro de la misma clase.  
📌 Explicación: Protege atributos y métodos internos.

### Pregunta 9
**¿Qué hace el modificador `protected`?**  
✅ Respuesta: Permite acceso en la clase y sus subclases.  
📌 Explicación: Facilita herencia manteniendo cierto nivel de protección.

### Pregunta 10
**¿Qué es una interfaz?**  
✅ Respuesta: Un contrato que define métodos obligatorios.  
📌 Explicación: Las clases que implementan la interfaz deben definir esos métodos.

### Pregunta 11
**¿Qué es una clase abstracta?**  
✅ Respuesta: Una clase que no puede instanciarse directamente.  
📌 Explicación: Sirve como base para otras clases.

### Pregunta 12
**¿Qué significa SRP?**  
✅ Respuesta: Single Responsibility Principle.  
📌 Explicación: Una clase debe tener una sola responsabilidad.

### Pregunta 13
**¿Qué significa OCP?**  
✅ Respuesta: Open/Closed Principle.  
📌 Explicación: El software debe estar abierto a extensión y cerrado a modificación.

### Pregunta 14
**¿Qué significa LSP?**  
✅ Respuesta: Liskov Substitution Principle.  
📌 Explicación: Las clases hijas deben poder sustituir a la clase padre sin errores.

### Pregunta 15
**¿Qué significa ISP?**  
✅ Respuesta: Interface Segregation Principle.  
📌 Explicación: Las clases no deben implementar métodos que no necesitan.

### Pregunta 16
**¿Qué significa DIP?**  
✅ Respuesta: Dependency Inversion Principle.  
📌 Explicación: Los módulos deben depender de abstracciones y no de implementaciones concretas.

### Pregunta 17
**¿Qué problema ocurre si una clase hija rompe el comportamiento esperado del padre?**  
✅ Respuesta: Se viola LSP.  
📌 Explicación: La subclase ya no puede sustituir correctamente a la clase base.

### Pregunta 18
**¿Qué relación UML representa herencia?**  
✅ Respuesta: Generalización.  
📌 Explicación: Representa una relación “ES UN”.

### Pregunta 19
**¿Qué relación UML representa agregación?**  
✅ Respuesta: Relación débil entre objetos independientes.  
📌 Explicación: Los objetos pueden existir por separado.

### Pregunta 20
**¿Qué relación UML representa composición?**  
✅ Respuesta: Relación fuerte de dependencia.  
📌 Explicación: Si el objeto principal desaparece, los objetos contenidos también.

### Pregunta 21
**¿Qué establece OCP?**  
✅ Respuesta: El software debe estar abierto a extensión y cerrado a modificación.  
📌 Explicación: Se agregan funcionalidades nuevas sin modificar código existente.

### Pregunta 22
**¿Qué problema generan interfaces demasiado grandes?**  
✅ Respuesta: Obligan a implementar métodos innecesarios.  
📌 Explicación: Esto viola ISP.

### Pregunta 23
**¿Qué ventaja ofrece DIP?**  
✅ Respuesta: Permite reemplazar implementaciones sin afectar módulos principales.  
📌 Explicación: Reduce acoplamiento y mejora flexibilidad.

### Pregunta 24
**¿Qué permite el polimorfismo?**  
✅ Respuesta: Que un método tenga diferentes comportamientos.  
📌 Explicación: Cada objeto responde de manera distinta al mismo mensaje.

### Pregunta 25
**¿Cuál es la diferencia entre agregación y composición?**  
✅ Respuesta: La composición implica dependencia fuerte del ciclo de vida.  
📌 Explicación: En agregación los objetos pueden existir independientemente.

### Pregunta 26
**¿Qué situación viola ISP?**  
✅ Respuesta: Obligar a implementar métodos innecesarios.  
📌 Explicación: Las interfaces deben ser específicas y pequeñas.

### Pregunta 27
**¿Qué ocurre si una subclase no puede reemplazar correctamente al padre?**  
✅ Respuesta: Se viola LSP.  
📌 Explicación: La herencia deja de ser válida funcionalmente.

### Pregunta 28
**¿Qué representa agregación?**  
✅ Respuesta: Una relación débil entre objetos independientes.  
📌 Explicación: Los objetos no dependen totalmente entre sí.

### Pregunta 29
**¿Qué diferencia existe entre interfaz y clase abstracta?**  
✅ Respuesta: La clase abstracta puede compartir estado y comportamiento parcial.  
📌 Explicación: La interfaz define principalmente contratos.

### Pregunta 30
**¿Qué busca el encapsulamiento?**  
✅ Respuesta: Ocultar implementación y proteger datos internos.  
📌 Explicación: Evita modificaciones indebidas desde el exterior.

## 🌐 Ingeniería Web, Atributos de Calidad y Metodologías

### Pregunta 31
**¿Qué atributo protege los datos frente a accesos no autorizados?**  
✅ Respuesta: Seguridad del sistema implementado  
📌 Explicación: La seguridad garantiza confidencialidad, integridad y disponibilidad de los datos.

### Pregunta 32
**Una empresa desarrolló un sistema correcto pero difícil de usar. ¿Qué atributo necesita mejorar?**  
✅ Respuesta: Usabilidad del sistema informático  
📌 Explicación: La usabilidad mide la facilidad de uso y eficiencia para que los usuarios realicen tareas.

### Pregunta 33
**¿Qué propiedad CSS activa Flexbox?**  
✅ Respuesta: display: flex;  
📌 Explicación: display: flex; convierte un contenedor en un modelo flexible para distribuir elementos.

### Pregunta 34
**¿Qué propiedad evita aumento de tamaño visual por padding?**  
✅ Respuesta: box-sizing: border-box;  
📌 Explicación: Hace que padding y border se incluyan dentro del ancho/alto total, sin aumentar el tamaño visual.

### Pregunta 35
**¿Qué modelo CSS usaría para distribuir elementos horizontalmente?**  
✅ Respuesta: Flexbox adaptable interfaz  
📌 Explicación: Flexbox está diseñado para alinear y distribuir elementos en una o varias dimensiones.

### Pregunta 36
**¿Qué selector usaría para reutilizar estilos en varios botones?**  
✅ Respuesta: Selector mediante clase  
📌 Explicación: Las clases permiten aplicar el mismo estilo a múltiples elementos sin repetir código.

### Pregunta 37
**Si index.html accede a logo.png en carpeta img, ¿qué ruta usaría?**  
✅ Respuesta: img/logo.png (o ./img/logo.png)  
📌 Explicación: La ruta relativa parte desde la ubicación del archivo actual (index.html).

### Pregunta 38
**¿Cuál etiqueta HTML define el contenido principal?**  
✅ Respuesta: main  
📌 Explicación: main representa el contenido central y único de la página, excluyendo cabeceras, pies, navegación.

### Pregunta 39
**¿Qué estrategia mejora SEO y accesibilidad?**  
✅ Respuesta: Etiquetas HTML semánticas  
📌 Explicación: Las etiquetas semánticas (header, nav, main, article) ayudan a motores de búsqueda y lectores de pantalla.

### Pregunta 40
**Un estudiante organiza un proyecto web profesional. ¿Cuál estructura es más apropiada?**  
✅ Respuesta: Separar css, js e img  
📌 Explicación: Separar en carpetas mejora mantenibilidad, orden y escalabilidad del proyecto.

### Pregunta 41
**Una empresa desea publicar información fija de contacto y ubicación sin interacción. ¿Qué tipo de sitio web sería más adecuado?**  
✅ Respuesta: Sitio estático con contenido informativo  
📌 Explicación: Los sitios estáticos no requieren bases de datos ni lógica del lado del servidor.

### Pregunta 42
**Una empresa desea incorporar videos sin plugins externos. ¿Qué tecnología usaría?**  
✅ Respuesta: Elementos multimedia HTML5  
📌 Explicación: Las etiquetas video y audio de HTML5 funcionan sin plugins como Flash.

### Pregunta 43
**Un sistema enviará contraseñas y datos personales. ¿Qué método HTTP usaría?**  
✅ Respuesta: POST para datos sensibles  
📌 Explicación: POST envía datos en el cuerpo de la petición, a diferencia de GET que los expone en la URL.

### Pregunta 44
**Un estudiante afirma que correo electrónico y videojuegos son servicios de la Web. ¿Cuál afirmación es correcta?**  
✅ Respuesta: La Web funciona sobre Internet  
📌 Explicación: La Web (HTTP/HTTPS) es un servicio más que se ejecuta sobre la infraestructura de Internet.

### Pregunta 45
**Un equipo mantiene reuniones frecuentes, valida avances y controla cambios. ¿Qué factor de éxito está aplicando?**  
✅ Respuesta: Comunicación efectiva del equipo  
📌 Explicación: La comunicación continua es clave para alinear expectativas y resolver problemas rápido.

### Pregunta 46
**¿Qué es SEO?**  
✅ Respuesta: Optimización para Motores de Búsqueda (Search Engine Optimization)  
📌 Explicación: Conjunto de técnicas para mejorar la posición de un sitio web en resultados de búsqueda.

### Pregunta 47
**Relaciona: DNS**  
✅ Respuesta: Traduce nombres a IP  
📌 Explicación: DNS convierte dominios como google.com en direcciones IP.

### Pregunta 48
**Relaciona: HTTP/HTTPS**  
✅ Respuesta: Protocolo transferencia  
📌 Explicación: Define cómo se comunican navegadores y servidores web.

### Pregunta 49
**Relaciona: Renderizado**  
✅ Respuesta: Construye página visual  
📌 Explicación: El navegador interpreta HTML/CSS/JS para dibujar la página en pantalla.

### Pregunta 50
**Relaciona: URL**  
✅ Respuesta: Dirección del recurso  
📌 Explicación: Localizador uniforme de recursos, ej: https://ejemplo.com/pagina

## 🗄️ Bases de Datos y Administración

### Pregunta 51
**¿Qué significa DDL?**  
✅ Respuesta: Data Definition Language  
📌 Explicación: Lenguaje para definir estructura de la BD (CREATE, ALTER, DROP).

### Pregunta 52
**¿Qué significa DML?**  
✅ Respuesta: Data Manipulation Language  
📌 Explicación: Lenguaje para manipular datos (SELECT, INSERT, UPDATE, DELETE).

### Pregunta 53
**¿Para qué sirve una PRIMARY KEY?**  
✅ Respuesta: Identificar de forma única cada fila en una tabla  
📌 Explicación: La PK no admite valores nulos ni duplicados.

### Pregunta 54
**¿Para qué sirve una FOREIGN KEY?**  
✅ Respuesta: Establecer relación entre dos tablas y mantener integridad referencial  
📌 Explicación: La FK apunta a una PK de otra tabla, evitando huérfanos.

### Pregunta 55
**¿Qué es un trigger?**  
✅ Respuesta: Procedimiento que se ejecuta automáticamente ante un evento (INSERT, UPDATE, DELETE)  
📌 Explicación: Útil para auditorías, validaciones o acciones automáticas.

### Pregunta 56
**¿Qué es la normalización?**  
✅ Respuesta: Proceso para eliminar redundancia y dependencias en una base de datos  
📌 Explicación: Reduce anomalías de inserción, actualización y borrado.

### Pregunta 57
**¿En qué forma normal se eliminan dependencias funcionales parciales?**  
✅ Respuesta: Segunda Forma Normal (2FN)  
📌 Explicación: Requiere estar en 1FN y que todos los atributos no clave dependan de la PK completa.

### Pregunta 58
**¿En qué forma normal se eliminan dependencias transitivas?**  
✅ Respuesta: Tercera Forma Normal (3FN)  
📌 Explicación: Un atributo no clave no puede depender de otro atributo no clave.

### Pregunta 59
**¿Qué es un índice en BBDD?**  
✅ Respuesta: Estructura que mejora la velocidad de búsqueda de datos  
📌 Explicación: Similar a un índice de libro; acelera SELECT pero ralentiza INSERT/UPDATE.

### Pregunta 60
**¿Qué significa ACID?**  
✅ Respuesta: Atomicidad, Consistencia, Aislamiento, Durabilidad  
📌 Explicación: Propiedades que garantizan transacciones confiables.

### Pregunta 61
**¿Qué propiedad ACID garantiza que una transacción se completa totalmente o no se realiza?**  
✅ Respuesta: Atomicidad  
📌 Explicación: Si algo falla, se deshace todo (ROLLBACK).

### Pregunta 62
**¿Qué propiedad ACID garantiza que la BD pase de un estado válido a otro válido?**  
✅ Respuesta: Consistencia  
📌 Explicación: Se cumplen reglas, restricciones y llaves.

### Pregunta 63
**¿Qué propiedad ACID garantiza que transacciones concurrentes no interfieran entre sí?**  
✅ Respuesta: Aislamiento (Isolation)  
📌 Explicación: Cada transacción ve la BD como si estuviera sola.

### Pregunta 64
**¿Qué propiedad ACID garantiza que una transacción confirmada persiste tras fallos?**  
✅ Respuesta: Durabilidad  
📌 Explicación: Los datos permanecen aunque se reinicie el sistema.

### Pregunta 65
**¿Qué es el transaction log?**  
✅ Respuesta: Registro de todas las transacciones para poder recuperar la BD ante fallos  
📌 Explicación: Permite restaurar hasta el último punto consistente.

## 📦 Modelos de Ciclo de Vida del Software (diferencias)

### Pregunta 66
**¿Cuál es la característica principal del modelo en cascada?**  
✅ Respuesta: Secuencial, requisitos fijos al inicio, no se puede volver atrás fácilmente  
📌 Explicación: Cada fase termina antes de empezar la siguiente.

### Pregunta 67
**¿Cuál es la característica principal del modelo en V?**  
✅ Respuesta: Cada fase de desarrollo tiene una fase de prueba asociada  
📌 Explicación: Énfasis en verificación (pruebas unitarias) y validación (pruebas de sistema).

### Pregunta 68
**¿Cuál es la característica principal del modelo espiral?**  
✅ Respuesta: Basado en riesgos, iterativo con evaluación de riesgos en cada vuelta  
📌 Explicación: Ideal para sistemas complejos y de alto riesgo.

### Pregunta 69
**¿Cuál es la característica principal del modelo incremental?**  
✅ Respuesta: Se entregan partes funcionales (incrementos) en iteraciones cortas  
📌 Explicación: Cada incremento añade funcionalidad al producto parcial.

### Pregunta 70
**¿Cuál es la principal diferencia entre cascada y modelo en V?**  
✅ Respuesta: El modelo en V añade pruebas asociadas a cada fase de desarrollo  
📌 Explicación: En cascada las pruebas son al final; en V son paralelas a cada etapa.

### Pregunta 71
**¿Cuál es la diferencia entre incremental e iterativo?**  
✅ Respuesta: Incremental entrega partes funcionales; iterativo refina el producto completo en cada ciclo  
📌 Explicación: Incremental = más funcionalidad; Iterativo = mejor calidad en lo existente.

### Pregunta 72
**¿Qué modelo de ciclo de vida es más adecuado para sistemas complejos y de alto riesgo?**  
✅ Respuesta: Espiral  
📌 Explicación: Porque evalúa riesgos explícitamente en cada iteración.

### Pregunta 73
**¿Qué modelo de ciclo de vida enfatiza la verificación y validación desde fases tempranas?**  
✅ Respuesta: Modelo en V  
📌 Explicación: Vincula cada fase de desarrollo con su fase de prueba correspondiente.

## 🧠 Ingeniería de Software y Requisitos

### Pregunta 74
**¿Qué es un requisito funcional?**  
✅ Respuesta: Describe lo que el sistema debe hacer (una funcionalidad específica)  
📌 Explicación: Ejemplo: "El sistema debe permitir iniciar sesión con correo y contraseña".

### Pregunta 75
**¿Qué es un requisito no funcional?**  
✅ Respuesta: Describe atributos de calidad como rendimiento, seguridad, usabilidad  
📌 Explicación: Ejemplo: "El sistema debe responder en menos de 2 segundos".

### Pregunta 76
**¿Qué norma internacional define los procesos del ciclo de vida del software?**  
✅ Respuesta: ISO/IEC 12207  
📌 Explicación: Establece procesos principales (adquisición, desarrollo, operación) y de soporte.

### Pregunta 77
**¿Cuál es la diferencia entre verificación y validación?**  
✅ Respuesta: Verificación = ¿construimos bien el producto? Validación = ¿construimos el producto correcto?  
📌 Explicación: Verificación revisa especificación; validación revisa necesidades del usuario.

## 🏗️ Arquitectura de Software

### Pregunta 78
**¿Qué es arquitectura de software?**  
✅ Respuesta: Estructura fundamental del software (componentes, relaciones, principios)  
📌 Explicación: Define la organización del sistema y sus decisiones técnicas clave.

### Pregunta 79
**¿Cuál es el rol del arquitecto de software?**  
✅ Respuesta: Define la estructura, toma decisiones técnicas, asegura atributos de calidad, guía al equipo  
📌 Explicación: Traduce requisitos en diseño técnico.

### Pregunta 80
**¿Qué son los drivers arquitectónicos?**  
✅ Respuesta: Requisitos que influyen fuertemente en la arquitectura (funcionales, calidad, restricciones)  
📌 Explicación: Ejemplo: alta concurrencia, seguridad extrema, bajo costo.

### Pregunta 81
**¿Qué norma define atributos de calidad del software?**  
✅ Respuesta: ISO/IEC 25010  
📌 Explicación: Sustituye a ISO 9126; incluye adecuación funcional, rendimiento, compatibilidad, usabilidad, fiabilidad, seguridad, mantenibilidad, portabilidad.

### Pregunta 82
**¿Qué es cohesión?**  
✅ Respuesta: Grado en que las responsabilidades de un módulo están relacionadas  
📌 Explicación: Alta cohesión = buen diseño; baja cohesión = módulo hace muchas cosas no relacionadas.

### Pregunta 83
**¿Qué es acoplamiento?**  
✅ Respuesta: Grado de dependencia entre módulos  
📌 Explicación: Bajo acoplamiento = cambios en un módulo afectan poco a otros.

### Pregunta 84
**¿Qué es escalabilidad?**  
✅ Respuesta: Capacidad de crecer en rendimiento o usuarios añadiendo recursos  
📌 Explicación: Escalar vertical (mejor hardware) u horizontal (más servidores).

### Pregunta 85
**¿Qué es tolerancia a fallos?**  
✅ Respuesta: Capacidad del sistema de seguir funcionando (quizás degradado) ante fallos  
📌 Explicación: Ejemplo: redundancia, reintentos, circuit breakers.

### Pregunta 86
**¿Qué es descentralización?**  
✅ Respuesta: Distribución de control sin un nodo central  
📌 Explicación: Ejemplo: blockchain, microservicios sin orquestador central.

### Pregunta 87
**¿Qué es interoperabilidad?**  
✅ Respuesta: Capacidad de intercambiar información y usar la información intercambiada  
📌 Explicación: Lograda mediante estándares, APIs bien definidas.

### Pregunta 88
**¿Qué caracteriza al patrón de capas (Layered)?**  
✅ Respuesta: Separación en capas (presentación, negocio, datos)  
📌 Explicación: Cada capa solo conoce a la inmediatamente inferior.

### Pregunta 89
**¿Qué caracteriza a los microservicios?**  
✅ Respuesta: Servicios pequeños, autónomos, desplegables independientemente  
📌 Explicación: Cada microservicio tiene su propia base de datos y ciclo de vida.

### Pregunta 90
**¿Qué es arquitectura Cloud Native?**  
✅ Respuesta: Diseñada para la nube (contenedores, CI/CD, resiliencia)  
📌 Explicación: Aprovecha servicios de nube como balanceadores, colas y almacenamiento distribuido.

### Pregunta 91
**¿Qué principios aplica la arquitectura segura?**  
✅ Respuesta: Defensa en profundidad, mínimo privilegio, seguridad por diseño  
📌 Explicación: La seguridad no es un añadido, se integra desde el diseño.

### Pregunta 92
**¿Qué es ATAM?**  
✅ Respuesta: Architecture Tradeoff Analysis Method – método para analizar riesgos y compensaciones  
📌 Explicación: Evalúa cómo los atributos de calidad compiten entre sí.

## 🔒 Seguridad Básica

### Pregunta 93
**¿Cuál es la diferencia entre autenticación y autorización?**  
✅ Respuesta: Autenticación = verificar identidad; Autorización = verificar permisos  
📌 Explicación: Primero autenticas (quién eres), luego autorizas (qué puedes hacer).

### Pregunta 94
**¿Qué es cifrado simétrico?**  
✅ Respuesta: Misma clave para cifrar y descifrar  
📌 Explicación: Rápido pero el desafío es compartir la clave de forma segura.

### Pregunta 95
**¿Qué es cifrado asimétrico?**  
✅ Respuesta: Par de claves (pública y privada)  
📌 Explicación: Lo que cifra una clave lo descifra la otra. La pública se comparte.

### Pregunta 96
**¿Qué es un hash?**  
✅ Respuesta: Función unidireccional que convierte datos en una huella fija  
📌 Explicación: No se puede revertir; útil para contraseñas y verificación de integridad.

### Pregunta 97
**¿Cómo se previene SQL Injection?**  
✅ Respuesta: Usar consultas parametrizadas (prepared statements)  
📌 Explicación: Separa la consulta de los datos, evitando que se interprete código malicioso.

### Pregunta 98
**¿Qué protocolo se usa para navegación segura?**  
✅ Respuesta: HTTPS (HTTP + SSL/TLS)  
📌 Explicación: Cifra la comunicación entre navegador y servidor.

### Pregunta 99
**¿Qué atributo de seguridad garantiza que los datos no sean modificados por no autorizados?**  
✅ Respuesta: Integridad  
📌 Explicación: Se logra con hashes, sumas de verificación, firmas digitales.

### Pregunta 100
**¿Qué atributo de seguridad garantiza que el sistema esté disponible cuando se necesita?**  
✅ Respuesta: Disponibilidad  
📌 Explicación: Se logra con redundancia, backups, balanceo de carga.
